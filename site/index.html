  • Anything you had to change or assume about this task to make your solution

    Data Emitter : Will emit the latest data received. Anyone listening to 'data-receive' event will get the latest data.

    Table Module: This will initialize and manage data emitter and html renderer.This will get the data from Data emitter and will parse it to html renderer.

    HTML renderer: This will be initialized by table module. It will get data from table module and render results to UI.

  • A high level description of your approach

    All the dependencies of table module have been injected to it while initializing.

    Observable design pattern is being used to emit the data received by data emitter.

    I have tried to follow the component based architecture approach using vanilla js. The solution consist of Forex class which uses the CurrencyPair class for each row. The Forex returns the new string template to be drawn as html each time the data is updated.

  • If you added any other development-time steps, what they are, why you added them and how to run them

    Added mocha and set up the unit testing. I have added unit test for one module. please run npm test to run the tests. It will also show the test coverage.
